# SoulMate Messenger - راهنمای نصب و راه‌اندازی

## 📋 پیش‌نیازها

- هاست اشتراکی با پشتیبانی از PHP 7.4 یا بالاتر
- پایگاه داده MySQL یا MariaDB
- فعال بودن افزونه PDO در PHP
- فعال بودن mod_rewrite در آپاچی (اختیاری)

## 🚀 مراحل نصب

### ۱. آپلود فایل‌ها
تمام فایل‌های پوشه `soulmate` را به هاست خود آپلود کنید:
```
public_html/soulmate/
├── index.php
├── config.php
├── database.sql
├── manifest.json
├── sw.js
├── api/
│   ├── auth.php
│   └── messages.php
├── assets/
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── app.js
│   └── images/
├── includes/
└── uploads/
```

### ۲. ساخت پایگاه داده

1. وارد phpMyAdmin شوید
2. یک پایگاه داده جدید بسازید (مثلاً: `soulmate_db`)
3. فایل `database.sql` را import کنید

### ۳. تنظیمات config.php

فایل `config.php` را ویرایش کرده و اطلاعات زیر را وارد کنید:

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'نام_پایگاه_داده');
define('DB_USER', 'نام_کاربری_پایگاه_داده');
define('DB_PASS', 'رمز_عبور_پایگاه_داده');
define('SITE_URL', 'https://yourdomain.com/soulmate');
```

### ۴. تنظیم دسترسی‌ها

دسترسی پوشه `uploads` را روی 755 تنظیم کنید:
```bash
chmod -R 755 uploads/
```

### ۵. دسترسی HTTPS (توصیه می‌شود)

برای امنیت بیشتر، SSL را فعال کنید تا آدرس سایت با `https://` شروع شود.

## 🔧 شخصی‌سازی

### تغییر کاربران

برای افزودن کاربران جدید، از پنل مدیریت phpMyAdmin استفاده کنید یا کوئری زیر را اجرا کنید:

```sql
INSERT INTO users (username, password_hash, display_name) 
VALUES ('username', '$2y$10$...', 'نام نمایشی');
```

برای ساخت هش رمز عبور جدید، از این کد PHP استفاده کنید:
```php
echo password_hash('رمز_عبور_جدید', PASSWORD_BCRYPT);
```

### تنظیم تاریخ شروع رابطه

در پنل تنظیمات هر کاربر، می‌توانید تاریخ شروع رابطه را مشخص کنید تا شمارشگر روزها کار کند.

## 📱 نصب به عنوان اپلیکیشن (PWA)

کاربران می‌توانند SoulMate را به صفحه اصلی گوشی اضافه کنند:

### اندروید (Chrome):
1. باز کردن سایت در Chrome
2. کلیک روی منوی سه نقطه
3. انتخاب "Add to Home screen"

### iOS (Safari):
1. باز کردن سایت در Safari
2. کلیک روی دکمه Share
3. انتخاب "Add to Home Screen"

## 🔔 نوتیفیکیشن‌ها

برای دریافت نوتیفیکیشن پیام‌های جدید:
1. اجازه دسترسی به نوتیفیکیشن را بدهید
2. تب را باز نگه دارید یا PWA را نصب کنید

## 🛡️ امنیت

### توصیه‌های امنیتی:

1. **HTTPS اجباری**: در Production حتماً از HTTPS استفاده کنید
2. **تغییر رمز پیش‌فرض**: رمز کاربران تست را تغییر دهید
3. **بک‌آپ منظم**: از دیتابیس بک‌آپ بگیرید
4. **به‌روزرسانی**: PHP و MySQL را به‌روز نگه دارید

### فایل .htaccess پیشنهادی:

```apache
# Force HTTPS
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# Protect sensitive files
<FilesMatch "^\.">
    Order allow,deny
    Deny from all
</FilesMatch>

# Prevent directory listing
Options -Indexes
```

## ❓ رفع مشکلات

### خطای اتصال به دیتابیس:
- اطلاعات config.php را بررسی کنید
- مطمئن شوید کاربر دیتابیس دسترسی دارد

### خطای 404 در API:
- مسیر فایل‌ها را بررسی کنید
- مطمئن شوید فایل‌ها با حروف کوچک/بزرگ صحیح آپلود شده‌اند

### پیام‌ها آپدیت نمی‌شوند:
- JavaScript کنسول را بررسی کنید
- مطمئن شوید polling interval درست کار می‌کند

### آپلود فایل کار نمی‌کند:
- دسترسی پوشه uploads را بررسی کنید (755)
- محدودیت upload_max_filesize در php.ini را بررسی کنید

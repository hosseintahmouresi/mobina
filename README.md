# SoulMate

فضای خصوصی تحت وب برای حسین و مبینا، ساخته شده برای هاست اشتراکی PHP/MySQL.

## قابلیت‌ها

- PWA نصب‌پذیر روی موبایل، تبلت و دسکتاپ
- ورود ماندگار روی دستگاه و ورود سریع با PIN
- عکس پروفایل اختصاصی برای هر نفر
- UI پیام‌رسانی مینیمال و لوکس با فونت Estedad
- چت دونفره realtime با polling سبک، مناسب هاست وردپرسی
- Web Push Notification با VAPID و Service Worker
- ارسال متن، عکس، ویس با پلیر اختصاصی، ویدئو، PDF، فایل متنی، Office، ZIP و RAR
- وضعیت ارسال، تحویل و خوانده‌شدن
- حالت تایپ کردن
- واکنش‌های عاشقانه، پیام قلبی، نامه عاشقانه، سوپرایز، ایده حرف، تعریف، قرار، چالش، خاطره، قول و روز ما
- ذخیره خودکار پیش‌نویس پیام روی همان دستگاه
- خاطره‌ها و تاریخ‌های مهم با نمایش و ورود تاریخ شمسی
- شمارنده روزهای کنار هم
- تم روشن/تاریک
- رمزگذاری متن پیام‌ها قبل از ذخیره در دیتابیس
- نصب‌کننده تحت وب برای ساخت جدول‌ها، دو حساب کاربری و کلیدهای Push

## نیازمندی‌ها

- PHP 8.0 یا بالاتر
- MySQL 5.7/MariaDB 10.3 یا بالاتر
- HTTPS فعال
- افزونه‌های PHP: `PDO MySQL`, `OpenSSL`, `JSON`, `fileinfo`
- برای Web Push بهتر است `cURL` فعال باشد
- فعال بودن آپلود فایل روی هاست

روی هاست‌های وردپرسی معمولاً این موارد از قبل فعال هستند. WebSocket لازم نیست.

## اجرای سریع

1. همه فایل‌ها را در یک پوشه مثل `public_html/mobina` آپلود کنید.
2. از کنترل‌پنل هاست یک دیتابیس و کاربر MySQL بسازید.
3. آدرس `https://your-domain.com/mobina/install.php` را باز کنید.
4. مشخصات دیتابیس، آدرس HTTPS برنامه، ایمیل VAPID و رمزهای حسین/مبینا را وارد کنید.
5. بعد از نصب، وارد برنامه شوید و روی هر دستگاه نوتیفیکیشن را فعال کنید.
6. بعد از اطمینان از نصب، فایل `install.php` را از هاست حذف کنید.

برای محیط واقعی، فایل‌های zip نسخه‌ها را در ریشه برنامه نگه ندارید. `.htaccess` پروژه دسترسی مستقیم به zip و فایل‌های مستندات را می‌بندد، اما حذف فایل‌های غیرضروری از هاست همچنان بهترین کار است.

اگر API خطای 500 داد، این آدرس را باز کنید:

```text
https://your-domain.com/mobina/api/health.php
```

این صفحه وضعیت PHP، افزونه‌ها، دیتابیس و جدول‌های اصلی را بدون نمایش رمزها بررسی می‌کند.

اگر حتی `health.php` هم باز نشد، این آدرس مستقل را باز کنید:

```text
https://your-domain.com/mobina/doctor.php
```

اگر `config/config.php` پاک شده بود، `repair.php` را باز کنید و مشخصات دیتابیس قبلی را وارد کنید. این مسیر داده‌های قبلی را حذف نمی‌کند:

```text
https://your-domain.com/mobina/repair.php
```

## آپدیت با یک فایل zip

برای نسخه‌های بعدی لازم نیست دیتابیس یا فایل‌ها را پاک کنید.

1. آدرس `upgrade.php` را باز کنید:

```text
https://your-domain.com/mobina/upgrade.php
```

2. فایل zip نسخه جدید را انتخاب کنید.
3. رمز ورود حسین یا مبینا را وارد کنید.
4. دکمه «آپلود و جایگزینی کامل» را بزنید.

این مسیر کدهای قبلی را جایگزین می‌کند، از نسخه قبلی در `storage/backups` بکاپ می‌گیرد و `config/config.php`، فایل‌های `uploads` و لاگ‌ها را حذف نمی‌کند. اگر فقط schema یا config نیاز به ترمیم داشت، هنوز می‌توانید `update.php` را باز کنید.

## نکته iPhone/iPad

روی iOS/iPadOS، Web Push برای وب‌اپ‌های اضافه‌شده به Home Screen از نسخه 16.4 به بعد پشتیبانی می‌شود. برای آیفون، اول وب‌اپ را به Home Screen اضافه کنید، بعد از آیکن نصب‌شده بازش کنید و نوتیفیکیشن را فعال کنید.

## بکاپ

برای بکاپ کامل، این سه مورد را نگه دارید:

- دیتابیس
- پوشه `uploads`
- فایل `config/config.php`

اگر `config/config.php` از بین برود، پیام‌های رمزگذاری‌شده قبلی دیگر قابل خواندن نیستند.

پیوست‌های پیام‌ها از مسیر احراز هویت‌شده `api/file.php` نمایش داده می‌شوند؛ مسیر مستقیم فایل‌های تصادفی داخل `uploads` برای پیوست‌ها قابل اتکا نیست و نباید در جای دیگری لینک شود.
